WP Google Maps Script

Script editing JSON data for entry into WP Google Maps Plugin

What I did

  • Used Python to develop application to transform JSON file coordinates into a format compatible with WP Google Maps

Project GitHub


Inspiration

For my work as a website technician, I was asked to test different solutions to implement a regional map function into WordPress websites. WP Google Maps was one of the first plugins I tested, but it was quickly ruled out as it was lacking multiple key features my team was searching for, including the ability to seemlessly interpret JSON coordinate file data. While inputting coordinate data to create a polygon region, WP Google Maps unconventionally swaps the longitude and latitude coordinates.


What it does

The WP Google Maps Script transforms traditional JSON file coordinates into a format compatible with the WordPress plugin, WP Google Maps, allowing users to quickly and efficiently add regions to their WordPress website. The script is designed for the "Polygon data" textbox entry under "Polygons" and can be run locally on a PC or online using Jupyter Notebooks.

How to use

If running the application locally, clone the project GitHub repository onto the PC and download the regional data JSON file into the same folder. If running the application online, open a Jupyter Notebook in Google Drive and upload the regional data JSON file into the same folder.

Read more below for guidance on obtaining JSON file data for map regions. Run transformCoordinates.py. Enter the JSON file name. The program will create a new .txt file with the text of the transformed coordinates, compatible with WP Google Maps. Close out of the running program, open the new .txt file, and copy its contents. Read more below on entering this data into the WP Google Maps Plugin and creating a polygon region.
